Support Process

Jump to:

  1. Check for the DID to be the same at doneness doc and global file (if RE, must be like the live site plus a 9 at the start)
  2. Set up the stats account - (https://stats.dealerspike.com/), (url, doneness New (http://bit.ly/newdoneness) - Old (http://bit.ly/olddoneness) & global, user, permission, auth key)
  3. Copy over events from the current site to the stage one
  4. Create the finance app header (follow stage site's design, include only the dealer logo and background elements)
  5. If the client has [newsletter account](https://myemma.com/login), create an updated stationary (copy on fred the psd files)
  6. Check for Staff Members, if so please add Staff page through Staff Builder
  7. NC: copy over testimonials (if there are pictures in the current site for testimonials, upload them to `/fckimages/testimonials`)
  8. RE: Add all Settings in **Dashboard Feature Management** as the live site

1. Check for the DID to be the same at doneness doc and global file (if RE, must be like the live site plus a 9 at the start)

  • Catches
  • Improvements

Global (New y RE):

  • Cuando el ID del global.asa no coincide con el doneness, ni viceversa. Se debio preguntar al launcher cual es el correcto.

(RE):

  • Cuando le falta un 9 al ID en el doneness, es decir que no esta actualizado. Aqui tendriamos que avisar al launcher.

2. Set up the stats account - (https://stats.dealerspike.com/), (url, doneness New (http://bit.ly/newdoneness) - Old (http://bit.ly/olddoneness) & global, user, permission, auth key)

  • Catches
  • Improvements

Global (New y RE):

  • Cuando en el global.asa no esta actualizado el ID de stats. La variable "TXT_DEALERSHIP_PIWIK_ANALYTICS_ID", esta vacia.
  • Si el sitio no tiene el permiso marcado en verde (columna "View").
  • Cuando el sitio es de Harley® y no esta agregado en la seccion de "Metasites"
  • Al ingresar al stats con el admin y pass del sitio, nos marca un error de "usuario y contraseña incorrectos".
  • Cuando en el stats del sitio aparecen varios sitios en el dropdown de "Tablero de un sitio especifico", que no son hermanos ni tienen una relacion aparente.

New Clients:

  • Si el nombre del sitio no coincide con el que esta en el doneness.
  • Si la url del sitio live no es igual al que esta en el doneness.
  • Si el usuario no esta creado. No hay resultados generados con el nombre del sitio.
  • Si en el doneness no esta el ID de stats generado, en la columna "Stats v2".
  • Si el correo electronico en el stats no tiene el formato adecuado al default (STATS@url-live.com)
  • Si el correo electronico no fue actualizado en el doneness.
  • Si el idioma en el stats del sitio esta en Español. Checar las configuraciones de la compu ya que a veces nos cambia el idioma automaticamente.
  • El API no fue ingresado en la opcion "Dashboard Feature Management" en el admin del sitio.

Redesign:

  • Si el correo electronico no coincide con el del doneness.
  • Ir a: Agregar toda la informacion de **Dashboard Feature Management** como en el sitio live.

New Clients:

  • --

Redesign:

  • Si el nombre del sitio no coincide con el que esta en el doneness. A veces los nombres estan diferentes, en el doneness, en el stats e incluso en la tarjeta de Trello. Aqui se buscaria un nombre adecuado, en donde se repita mas, o donde sea coherente.
  • Si la url del sitio live esta diferente al doneness, o viceversa, actualizarlo de preferencia para que se registre la misma informacion.

3. Copy over events from the current site to the stage one

  • Catches
  • Improvements

(New):

  • Si no se pasaron todos los eventos, desde la fecha actual.
  • Si a los eventos les falta algun dato del sitio live (descripción e imagen) que se puedan agregar.
  • Si el sitio es powersports y en tipo de categoria tiene otro que no sea "Rides".
  • Si el sitio no es powersports y tiene como tipo de categoria "Rides", este deberia tener el tipo "Events".

(New):

  • Cuando las imagenes no se pueden agregar a un evento. Ya sea porque estan muy grandes o por alguna otra situacion, se pediria respaldarlas.

4. Create the finance app header (follow stage site's design, include only the dealer logo and background elements)

  • Catches
  • Improvements

Global (New y RE):

  • Si el header no aparece, ya que el nombre puede estar escrito incorrectamente. No confundir cuando aun no se activa el financeApp. En este caso podemos ir al servidor live y verificar si la imagen de header se encuentra en la carpeta "/imglib/FinanceApp/images".
  • En la mayoria de los casos, el logo debe estar centrado, respetando tambien el background del sitio.
  • Si el PSD del header del financeApp no esta respaldado en el servidor FRED.

Global (New y RE):

  • Agregar una barra que simule que el navbar esta debajo, por apariencia se ve mejor, pero tomar en cuenta que no todas las veces es posible ya que no todos los navs estan debajo del header, si no estan dentro del header.
  • Para organizar mejor el trabajo se pide que la carpeta donde respaldemos el PSD, tenga el nombre con el año y el mes dentro de la carpeta del sitio.
  • Cuando se requiere actualizar el logo, ya que en el trancurso fue cambiado por otro.

5. If the client has [newsletter account](https://myemma.com/login), create an updated stationary (copy on fred the psd files)

  • Catches
  • Improvements

Global (New y RE):

  • Cuando hay cuenta y el template no esta creado.
  • Cuando el template creado no tiene nombre (nombre del sitio, guiarnos por el doneness).
  • El header del newsletter debe tener el logo, direccion y telefono del sitio.
  • El template debe tener linkeado el header al sitio live.
  • El template debe tener la url del live agregado en el footer. Ultimamente hay problemas para linkear, omitir este paso hasta nuevo aviso.
  • Si el template no se comparte a la cuenta del dealer.
  • Si el PSD del header del newsletter no esta respaldado en el servidor FRED.

Global (New y RE):

  • Para organizar mejor el trabajo se pide que la carpeta donde respaldemos el PSD, tenga el nombre con el año y el mes dentro de la carpeta del sitio.
  • Respaldar el archivo del header de newsletter con la extension del editor de imagenes "Gimp", que se usa actualmente, para facilitar la edicion del mismo dentro del departamento. Igualmente crear una carpeta con el mes y año actual.

New Clients:

  • Crear el template aun cuando no existe la cuenta, ya que el launcher puede crear la cuenta durante el proceso, tendriamos esa ventaja de tener el template ya listo solamente para compartir.

6. Check for Staff Members, if so please add Staff page through Staff Builder

  • Catches
  • Improvements

Global (New y RE):

  • Cuando hay xstaff y no esta linkeado al navbar.
  • Cuando falta informacion o es incorrecta en los miembros del staff que estan en el live.
  • Cuando los miembros del staff no estan organizados jerarquicamente dentro de su departamento.
  • Cuando hay informacion aparte del staff, se debe agregar ya sea en el snippet que esta en el header de xstaff (cuando es poca información), o crear una pagina que se llame staff y poner un boton linkeado a xstaff (cuando es mucha informacion).

Redesign:

  • La regla en RE es copiar todas las paginas linkeadas en el sitio. Existen paginas que solamente tienen a ciertos departamentos del staff, con o sin informacion, estas paginas deben copiarse y mandar a llamar el snippet con el script del xstaff.

Global (New y RE):

  • Cuando no hay xstaff, podemos linkear el xstaff que viene por default a coming soon. Para evitar que la pagina de xstaff se vea vacia, sin contenido.
  • Cuando existen dos telefonos o tiene dos puestos algun miembro del staff, pero solo se puede capturar uno en la herramienta. Se pide agregarlo en la bio, para que la informacion este completa.

NC: copy over testimonials (if there are pictures in the current site for testimonials, upload them to `/fckimages/testimonials`)

RE: Add all Settings in **Dashboard Feature Management** as the live site
